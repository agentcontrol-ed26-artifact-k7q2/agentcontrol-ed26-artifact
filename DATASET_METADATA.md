# DATASET_METADATA

Metadata describing AgentControl's data assets, intended for NeurIPS 2026 Evaluations & Datasets reviewers and Croissant consumers.

## What AgentControl is submitted as

**All three** of:

1. an **evaluation harness** (Python source under `src/`, `scripts/`, `tests/`);
2. a **set of synthetic-local task pools** with deterministic gold answers and verifiers (under `src/agentcontrol/tasks_*.py`, `src/agentcontrol/hard_regime_tasks.py`, `src/agentcontrol/rescue_tasks.py`, `method_discovery/`);
3. a **cached real-API trace + cost-ledger artifact** (under `cache/`, `traces/`, `experiments/`).

The paper's contribution is the **methodology** that ties all three together (fair-baseline regime mapping under three rules). The data assets exist to make the methodology and its three falsifications reproducible.

## Task-pool provenance

All task pools are **synthetic-local**: programmatically generated by code in this repository. None are drawn from MATH-500, GPQA, HumanEval+, MBPP+, AIME, or any standard benchmark.

| pool | n | family | regime / subset | source code |
|---|---|---|---|---|
| smoke math/code | 28 | math, code | various | `src/agentcontrol/tasks_math.py`, `src/agentcontrol/tasks_code.py` |
| smoke evidence | varies | evidence | various | `src/agentcontrol/tasks_evidence.py` |
| rescue | 60 | math, code, evidence | mixed | `src/agentcontrol/rescue_tasks.py` |
| hard regime | 90 | math, code, evidence | 4 sub-regimes per family | `src/agentcontrol/hard_regime_tasks.py` |
| interactive (sprint 6) | 120 | math+checkpoints, code, evidence, data_analysis, tool_planning | various | `main_rescue_gpu/src/agentcontrol_main_rescue/interactive_tasks.py` |
| F1 agentic search | 60 | evidence (multi-hop) | controlled multi-hop closed corpus with deliberate distractors | `method_discovery/scripts/build_agentic_search_pool.py` |
| F2 data_analysis | 50 | data_analysis_code | F2 falsification pool | `method_discovery/scripts/build_f2_data_analysis_pool.py` |
| Apollo verifier-risk | 60 | evidence-style | adversarial weak-verifier pool | `main_apollo/verifier_risk/scripts/build_verifier_risk_pool.py` |
| Apollo A-GPU label set | 388 | mined from cached traces (sprints 1-8) | for label-readout pre-screen | `main_apollo/kv_readout/scripts/build_a_gpu_readout_dataset.py` |

For every task: `task_id`, `family`, `regime`, `question`, `answer` / `gold_phrases`, `authorized_citation_set` (where applicable), and verifier function are persisted in the source code. Generation is deterministic; distractor selection in evidence pools uses `md5` (process-stable, independent of `PYTHONHASHSEED`).

## Traces

`traces/*.jsonl` records, for every (task, action) pair attempted across the nine sprints, one row containing `{task_id, family, regime, action, success, cost_usd, input_tokens, output_tokens, latency_ms, cache_hit, actual_api_call, request_hash, provider, model, timestamp}`. Traces are fully replayable against the cache.

## Cached LLM responses

`cache/provider/*.json` contains every real-API response collected across the project (2,400+ payloads, ≈ 30 MB). Each file is named by the SHA256 of its canonical request body. Each payload preserves the upstream provider's `model` field. Release terms: see `LICENSE` §3 (academic reproducibility, subject to provider ToS; no commercial redistribution; no model-training reuse).

## Cost ledger

`cache/cost_ledger.jsonl` is append-only and tracks every real-API call: `{request_hash, provider, model, input_tokens, output_tokens, cost_usd, latency_ms, cache_hit, actual_api_call, timestamp}`. Cumulative across nine sprints: 2,400+ rows; **$0.65** real spend; $0 incremental on second-pass replay. Reconciliation manifest at `experiments/hard_regime_ledger_manifest.json`.

## Limitations (data-side)

See `paper/ed_final_limitations.md` for the canonical limitations list. Highlights specific to data assets:

- Synthetic-local task pools have **reduced external validity**; the protocol generalizes, the specific labels do not.
- Per-cell sample size is small (4–30 typically).
- Evaluated providers are two: DeepSeek and Together AI. Frontier-tier models (GPT-5, Claude Opus / Sonnet, Gemini Pro) are not evaluated.
- Verifier strictness varies by family: math = numeric exact match on last integer; code = in-process Python `exec` + unit tests (no wallclock timeout); evidence = gold-phrase substring + ≥ 1 authorized citation.
- The Apollo verifier-risk pool is hand-engineered, not benchmark-derived.

## Intended use

- **Academic review-time reproducibility** of the AgentControl paper's reported numbers ($0 spend, 0 GPU hours).
- **Methodology re-use**: applying the three fair-baseline rules to a new task pool / provider mix using this harness as a template.
- **Negative-result reference**: citing the three falsifications when assessing future orchestration claims.

## Prohibited use

- Commercial redistribution of cached LLM responses.
- Use of cached LLM responses as training data, fine-tuning data, or distillation data.
- Any redistribution that conflicts with the upstream providers' then-current Terms of Service.
- Use as a benchmark against which to claim SOTA on a generic task family — the pools are synthetic-local and were constructed to expose specific evaluation patterns, not to rank models.

## Responsible AI notes

- No human subjects; no PII; no consent or IRB applicable.
- Synthetic tasks contain no copyrighted source material outside fair use.
- Cached LLM responses are model-generated answers to math / code / evidence-citation / data-analysis tasks; spot-checked for offensive content (clean).
- The artifact does NOT recommend any deployment policy. The contribution is methodological. See `paper/ed_final_broader_impact.md`.
- Evaluated providers may have undocumented serving non-determinism; cache replay holds the seed-equivalent state at collection time. This is a feature for reviewer reproducibility but a limitation for distribution-shift studies.
